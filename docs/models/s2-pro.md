# S2-Pro

这份文档从“私人智能助手”场景出发解释 `s2-pro-4b-gguf`。

## 先说结论

如果硬件满足，`s2-pro-4b-gguf` 应该是 Agent Voice 的第一优先梯队。

原因很直接：

- 它更适合追求情感表达
- 它更适合追求拟人感
- 它更适合把“当前这句话应该怎么说”做得更自然

对私人智能助手来说，这比多人对话、多参考这些能力更重要。

## Agent 应该怎么选它

Agent 不需要先纠结一堆参数，而是先判断两件事：

1. 当前机器显存和内存是否够
2. 当前机器推理效率是否足够满足实时语音生成

如果都满足，就优先考虑 `s2-pro-4b-gguf`。

## Agent 真正需要关注什么

对 Agent Voice 来说，S2-Pro 几乎不需要关注额外参数。

不需要 Agent 重点关注：

- `speaker`
- `temperature`
- `top_p`
- `top_k`
- `max_length`
- `min_length`

也不需要把多人对话、多参考当成主工作流。

在当前项目里，Agent 最应该关注的是：

- 这句话要不要做风格控制
- 这句话的推理文本应该怎么写，才能更像真人在说
- 如果要控制风格，是否应该直接写一个显式内联标签

## 正确的使用重点

S2-Pro 的关键不是“调参”，而是“给对推理文本”。

也就是说，Agent 要重点决定的是：

- 正文是不是足够自然
- 正文是不是符合私人助手口吻
- 是否需要风格控制
- 风格控制是只靠正文表达，还是直接写一个显式标签

## 文本写法建议

### 1. 先写像人说的话

不要把文本写成机械通知。

更推荐：

```text
你刚刚那份文档我已经看完了，核心问题主要有两个，我现在先把最重要的一个讲给你。
```

不推荐：

```text
文档已处理完成。现开始输出问题一。
```

### 2. 风格控制优先显式写成内联标签

这一点需要明确写出来：

S2-Pro 的风格语言标签服从能力比较强。对 Agent Voice 来说，如果你已经知道自己要的情绪、语气或发声方式，直接写成显式 `[tag]`，通常会比只在正文外侧做模糊描述更容易被模型稳定遵守。

它支持两类思路：

- 常见显式标签
- 自由文本标签

自由文本标签示例：

```text
[whisper in small voice]
```

```text
[professional broadcast tone]
```

```text
[pitch up]
```

同时它也支持大量常见标签。对 Agent Voice 来说，下面这组明确写出来的标签，通常更适合作为第一选择：

- 停顿与强调：
  - `[pause]`
  - `[short pause]`
  - `[emphasis]`
- 呼吸与口腔动作：
  - `[inhale]`
  - `[exhale]`
  - `[clearing throat]`
  - `[panting]`
- 笑与轻松表达：
  - `[laughing]`
  - `[laughing tone]`
  - `[chuckle]`
  - `[chuckling]`
- 情绪与态度：
  - `[excited]`
  - `[excited tone]`
  - `[delight]`
  - `[surprised]`
  - `[shocked]`
  - `[sad]`
  - `[angry]`
  - `[sigh]`
- 音量与音高：
  - `[volume up]`
  - `[volume down]`
  - `[low volume]`
  - `[low voice]`
  - `[loud]`
- 说话方式：
  - `[whisper]`
  - `[singing]`
  - `[interrupting]`
  - `[with strong accent]`
  - `[echo]`
  - `[screaming]`
  - `[shouting]`
- 其他氛围类：
  - `[tsk]`
  - `[audience laughter]`
  - `[moaning]`

推荐写法示例：

```text
[calm] 这个问题先别着急，我已经帮你把最关键的风险点找出来了。
```

```text
[gentle] 我记得你上次更偏向第二个方案，这次我先按那个方向继续讲。
```

```text
[serious] 这里有一个会直接影响线上行为的问题，我建议你先处理它。
```

```text
[whisper in small voice] 先别打断我，我把这个地方讲完你就知道为什么它危险了。
```

```text
[excited tone] 这次是真的解决了，不是临时绕过去的那种修补。
```

### 3. 标签服从更强，但不要单句堆太多

“显式标签服从更强”不等于“可以无限堆标签”。

对私人智能助手场景，最稳的做法仍然是：

- 一句话最多点 1 到 2 个关键标签
- 优先选最贴近当前说话意图的标签
- 不要把整句前后塞满风格描述
- 如果只是轻微情绪变化，用一个明确标签通常就够

### 4. 不要把它当多人对话模型来用

虽然 S2-Pro 有 speaker tags 和多说话人能力，但在 Agent Voice 这个场景里，这些都不是重点。

这里最重要的是：

- 单人助手声音是否自然
- 语气是否贴合上下文
- 是否能听起来不像模板播报

## 推荐的心智模型

对 S2-Pro，Agent 最好这样想：

- 不是“我要传什么参数”
- 而是“我要把这句话改写成怎样的人类说法，以及要不要加一个明确的内联标签”

## 已知约束

- 当前 Runtime 只公开单参考角色注册
- 多参考对 Agent Voice 不是主路径
- 真正影响效果的重点不在参数，而在推理文本和显式内联风格控制
